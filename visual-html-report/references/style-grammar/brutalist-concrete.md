# 粗野水泥

## 一句话语法

粗野主义 · 水泥灰 · 全部等宽 · 浏览器默认蓝下划线 · 视觉粗粝

## 适合场景

- 技术审计
- 开发者生态报告
- 基础设施报告

## 组合规则

- 让内容决定模块；这个风格只控制气质、密度和视觉处理。
- 除非该风格迁移到其他引擎，否则保持报告源码包结构不变。
- 在生成包中，把选中的主题 CSS 作为 `styles/theme.css` 使用。
- 根据该风格调整标题、KPI 密度、表格、图表颜色和编辑型模块。
- 不要强迫每份报告模仿展厅顺序。

## 模块倾向

- 排名、矩阵、直方图、时间线和洞察模块通常安全。
- 套用该主题后，密集表格必须检查对比度。
- 对视觉感很强的风格，优先使用少量更有力的模块，不要堆很多小装饰章节。

## 避免

- 随机装饰性图表
- 没有标签和单位的可视化
- 机械复制展厅章节顺序
- 柔和 SaaS 卡片和亲和粉彩插画
