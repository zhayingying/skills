单看 star 数容易被刷量误导。Fork 数代表真正"想要改/部署/学习"代码的开发者数量，是更扎实的活跃度指标。圆圈大小 = 项目体量综合得分。

§ 03 · 家族关系

生态系谱：谁 fork 自谁

现代 coding CLI 的高密度繁殖期是 2024 底 ~ 2026 初。一个家族经常一年内冒出三代：Cline → Roo Code → Kilo Code；Gemini CLI → Qwen Code。Claude Code 没有直接 fork，但 OpenCode / Crush / Goose 都是它定义的"终端 + plan/act + MCP"模式的变体。

家族关系备忘：Cline 原名 Claude Dev，2024 年底改名；Roo Code 是 Cline 的 fork，加了 multi-mode 系统；Kilo Code 又从 Roo fork，加了 JetBrains 支持。Roo Code 团队 2026年4月21日宣布 5月15日停服，全力转型做云端 agent (Roomote)。Qwen Code 是阿里完全 fork 自 Google 的 Gemini CLI，深度适配 Qwen3-Coder。OpenHands 前身就是大名鼎鼎的 OpenDevin。

§ 08 · 解读

五个值得记住的规律

No. 01

终端正在反扑 IDE

2025 年 Cursor 一度让所有人以为编程的未来在浏览器 IDE。但 2025 年中开始，Claude Code、Codex CLI、Gemini CLI、OpenCode 接连发布，让 CLI 形态拿下了 star 数的前 6 个席位之中的 5 个。原因很简单：CLI 进 CI/SSH 远程、批处理、headless 模式都不需要妥协。

No. 02

MCP 已成事实标准

Anthropic 2024 提出的 Model Context Protocol 现在被 OpenAI、Google、微软全部采纳。16 个表内项目里 13 个原生支持 MCP。MCP 已经从 Anthropic 私货变成了 AI Agent 间集成的"USB-C 标准"。2026 年 4 月，连同 AGENTS.md 和 Goose 一起，被捐给了 Linux Foundation 旗下的 AAIF。

No. 03

Rust + TS 的双语言生态

注意 Codex / Goose / Tabby 都用 Rust 重写（甚至原始就是 Rust），核心诉求是：可分发单二进制、启动毫秒级。同时 TypeScript 阵营（OpenCode / Cline / Continue / Gemini CLI）则吃 VS Code 和 Node 生态。Python 退到了"研究框架 + 教育"位置（SWE-agent、Aider、OpenHands）。

No. 04

Fork 比 Star 更说明问题

CMU 的 ICSE 2026 论文表明 GitHub 有 600 万颗虚假 star，AI/LLM 类是最大重灾区。看哪些项目 fork 数和 star 数比例失调（如 OpenCode 18.8k fork / 160k star = 11.7%，而正常开源项目通常 5–8%）可能反映出真实的工程师社区规模。bolt.diy 的 fork 比异常高（>25%），因为它本身鼓励自建。

No. 05

洗牌已经开始

Roo Code（24k star）2026年5月15日全面停服，团队转型 Roomote 云端 agent，宣称"IDE 不是编程的未来"。Block 把 Goose 捐给 Linux 基金会。Cursor、Windsurf、Replit Agent 4 等商业产品也在重新洗牌。开源 CLI 这一波很可能在 2026 下半年继续收敛——但终端原生的形态本身会留下来。

No. 06

你听过的"百万 star"项目可能不存在

本次调研中我刻意核查了某些 SEO 文章里出现的"Claw Code 191k star"、"OpenClaw 371k star"、"Hermes Agent 144k star"——它们在 GitHub 上根本搜不到。LLM 生成的低质量内容污染了搜索结果。本表所有数据均来自 GitHub repo 本体的当前 star 数（截至 2026-05）。

§ 09 · 你问到的

Pi Agent · 你点名问到的"小而美"

由 libGDX 之父 Mario Zechner (badlogic) 一人写的极简 coding harness。星数不到 2k，但代码量极小、上下文工程能力最完整、扩展模型最干净。是上面任何一个的"原教旨主义"版本。

核心理念

Pi 默认只给模型 4 个工具：read / write / edit / bash。 哲学是"如果 AI 能用好 bash，它就能做任何事"。没有 plan mode、没有子 agent、没有 fancy 集成——所有这些都通过 Skills、Extensions、Prompt Templates、Pi Packages 用户自己装。差异化渲染（diff-rendering TUI）保证终端不闪烁。

对照其他工具

代码行数~10k
核心工具数4
Provider 支持所有主流 + 自定义
扩展机制npm / git 包
作者badlogic (1 人)
类比：如果 Claude Code 是 macOS（精致一体），那 Pi 就是 Linux （拼装 + 极简）。如果你写终端工具时被那些大型 agent "吃太多 token 又不让我控制"激怒过，Pi 可能就是你要找的东西。它的设计哲学和 Aider 相通但更激进。

和弦图：工具之间的"血亲度"

柱状条按值降序排列 + 一条累计百分比折线。经典"80/20 分析"——能看出"前几个工具贡献了 80% 的 star 数"。这里看：头 4 个工具占整个生态 53% 的 star。

§ 43 · 小宇宙阵列

小图阵列（Small Multiples）

同样的图表样式重复 N 遍，每个对应一个类目。读者可以一眼对比模式。这种格式比单个大图能容纳更多信息，而且大脑读取速度反而更快。

§ 44 · 阶跃

阶梯图：Claude Code 版本里程碑

§ 46 · 新闻段落

报刊式开场段（Newspaper Lede）

最简单也最具仪式感的排版——首字母放大、首行小型大写字母。适合放在报告的开头作为"叙事入口"。

在 2026 年 5 月这个时间点回望，开源编程 Agent 的过去十二个月经历了一场惊人的洗牌。Cursor 与 Windsurf 作为商业 IDE 的代表，一度被认为代表了"AI 编程的未来形态"。但 Anthropic 在 2025 年 2 月发布的 Claude Code 重新定义了这场游戏——它选择了一条没人押注的路：纯命令行、不做 IDE、不做扩展。结果是十二个月内 123k stars、3M 周活，并直接催生了 OpenCode、Crush、Goose、Pi 等一整代"终端原生"开源工具。这份调研记录了这场迁移的核心数据。
§ 47 · 多栏排版

三栏报刊（Three Column Newspaper）

把大段文字切成三栏，模拟纸质报纸的视觉密度。当一段文字超过 400 字、需要传达"信息密度"感时使用。

方法论 · 关于这份调研

这份报告的所有数据点都来自对 GitHub 仓库本体的直接核查——而不是来自 SEO 文章、不是来自 LLM 生成的二手内容、也不是来自任何"工具汇总站"。每一项 star 数都是 2026 年 5 月 15 日打开对应 repo 页面读取的当前值。每一项 fork 关系都通过 README、commit history、Hacker News 讨论或 Wikipedia 条目交叉核对。

这并不是过度严格——而是必要。我们在调研过程中发现，某些 SEO 文章里反复出现的"百万 star 项目"（如 "Claw Code 191k"、"OpenClaw 371k"、"Hermes Agent 144k"）在 GitHub 上完全搜不到。这些项目根本不存在。它们是 LLM 在生成内容时凭空虚构、然后被 SEO agency 大量复制传播的。

这背后是更深的问题。CMU 在 2026 年 ICSE 上发表的一篇论文表明，GitHub 上至少有 600 万颗虚假 stars，AI/LLM 相关项目是重灾区。某些项目通过付费"刷星服务"在发布首周内造势冲榜。识别真实流行度需要看 fork/star 比例、issue 数、commit 频率等多维信号。

所以本报告把 fork 数作为同等重要的指标和 star 数并列展示。在 §02 的散点图和 §23 的哑铃图里都能看出，几个工具的 fork/star 比例显著偏离均值。这通常意味着两种情况：要么真实开发者社区比 star 暗示的更深（如 bolt.diy），要么 star 被刷过（无法点名，但你可以自己看图判断）。

最后说一句：本报告刻意不收录商业闭源工具（Cursor、Windsurf、Devin、Replit Agent）。这不是说它们不重要——而是这份报告聚焦的是"开源、可自部署、可 fork"的工具生态。商业产品适合另起一份报告专门讨论。
